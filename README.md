# grad_2026_hack_g

アイドル × リズム × ターン制RPG。現実の歩行距離と連動し、都市に出現する謎の怪物 **Noise** を討伐して街を解放していく Unity モバイルゲームです。

---

## 作品概要

人々はアイドルとして活動しながら、都市に出現する謎の怪物「Noise」と戦う使命を背負う。音楽ゲームのリズム要素とターン制RPGを融合させたバトルシステムを採用し、楽曲に合わせた操作によってパフォーマンスを高めながら戦闘を進めていく。プレイヤーは各地で発生する Noise を討伐し、人々の希望を取り戻しながら、蝕まれた都市全体の解放を目指す。

---

## 遊び方

### 全体の流れ

```
タイトル → ホーム →（スキャン）→ ステージ選択 → 装備 → バトル → リザルト → ホーム
                ↘ クラフト（装備合成）
```

1. **タイトル（TitleScene）**  
   ゲームを開始し、ホーム画面へ進みます。

2. **ホーム（HomeScene）**  
   拠点画面です。歩いた距離（GPS）に応じて **NoisesAmount** ゲージが増え、地図上に出現する Noise の数が決まります。  
   主なメニュー：
   - **スキャン** — 地図上の Noise をスキャンして位置を確認（1日1回）
   - **ステージ** — 討伐する Noise を選択
   - **装備** — 戦闘前のコスチューム・装備を変更
   - **クラフト** — バトル報酬素材から Top / Bottom / CD を合成

3. **スキャン（ScanScene）**  
   その日初めてホームから進むときのみ表示されます。スキャンボタンで地図を走査し、未発見の Noise が順次表示されます。確認後、ステージ画面へ進みます。  
   （2回目以降はホームから直接ステージへ遷移します。）

4. **ステージ（StageScene）**  
   CD ターンテーブルを回しながら、出現済みの Noise 一覧から討伐対象を選びます。  
   - Noise は最大 12 体（SuperRare 1 / Rare 1 / Normal 10）
   - 歩行距離に応じて上から順に解放（SR → Rare → Normal…）
   - 撃破済みの Noise はリストから消えます  
   **Ready** で出撃準備 → **Battle** でバトルへ。

5. **装備（OutfitScene）**  
   ステージ画面やホームからアクセス可能。装備はプレイヤーのステータス（HP など）と見た目に反映されます。

6. **クラフト（CraftScene）**  
   バトル報酬（SuperRare / Rare / Normal）を素材に、Top・Bottom・CD の装備を合成します。作成後はそのまま装備することもできます。

7. **バトル（MainScene）**  
   浄化バトル本体。ターン制コマンドバトルにリズム QTE が融合した 2D 戦闘です（詳細は下記）。

8. **リザルト（ResultScene）**  
   勝敗結果と報酬を表示。勝利すると歩行距離ゲージが減算され、次の Noise 解放に向けて再び歩くモチベーションが生まれます。

### 歩行距離と Noise 解放

- 端末の **GPS** で歩いた距離を計測し、累計距離から **NoisesAmount** を算出します（例：5,000 m ごとに +1、最大 12）。
- 日次スケジュールが有効な場合、初日は一定の距離が付与され、翌日以降は前日の歩行分が反映されます。
- 距離が増えるほど多くの Noise がマップ・ステージ一覧に出現し、討伐対象が増えていきます。

## バトルシステム

### 基本ルール

| 項目 | 内容 |
|------|------|
| 形式 | ターン制コマンドバトル ＋ 太鼓風リズム QTE |
| ウェーブ | 1 ステージあたり最大 3 ウェーブ（敵 1〜3 体／ウェーブ） |
| コマンド | **攻撃**（単体）／ **回復**（自身） |
| 行動順 | プレイヤーと敵の **Speed** で決定（同速はランダム） |
| HP | ウェーブをまたいで引き継ぎ。0 で敗北 |

### プレイヤーターンの操作

1. **攻撃** または **回復** を選択（2 段階入力：1 回目で選択 → 2 回目で確定）
2. 攻撃の場合は **ターゲット** をタップして敵を指定
3. **QTE フェーズ** — 楽曲に合わせて流れる音符をタイミングよくタップ（または **Space** キー）
4. 判定（Perfect / Good / Miss）に応じた倍率でダメージ・回復量が決定

### QTE と倍率

- 各スキルは最大 5 ノートのリズム判定
- **Perfect** ×1.3、**Good** ×1.1、**Miss** は Perfect を打ち消し
- **全ノート Perfect（AP）** のとき：
  - 攻撃 → 生存全敵に同ダメージを同時適用
  - 回復 → 追加倍率（×2）で大回復

### 敵の行動

- 生存している敵がまとめて攻撃（闇の球 Trail → プレイヤーへ）
- ラウンドが進むと **狂暴化**（3 / 5 / 7 ラウンド目に攻撃力上昇、最大 3 段階）

### 勝敗

- **勝利** — 全ウェーブの敵を撃破 → リザルトへ（報酬・距離減算）
- **敗北** — プレイヤー HP が 0 → リザルトへ

チュートリアルは **InGameTutorial** シーンで体験できます。

## プロジェクト構成

```
grad_2026_hack_g/
└── HackathonGame/          # Unity プロジェクト本体
    ├── Assets/
    │   ├── InGame/         # バトル（浄化戦闘）のロジック・アセット
    │   ├── OutGame/        # ホーム・スキャン・ステージ・装備・クラフト
    │   ├── Shared/         # シーン遷移・共通ユーティリティ
    │   └── Scenes/         # 各シーンファイル
    └── Docs/               # 仕様書・開発ドキュメント
        ├── InGameSpec.md   # バトル詳細仕様
        └── ProjectStructure.md
```

### 主なシーン

| シーン | 役割 |
|--------|------|
| TitleScene | タイトル |
| HomeScene | ホーム拠点 |
| ScanScene | 地図スキャン |
| StageScene | Noise 選択・出撃 |
| OutfitScene | 装備変更 |
| CraftScene | 装備合成 |
| MainScene | 浄化バトル |
| InGameTutorial | バトルチュートリアル |

## 開発環境

| 項目 | バージョン |
|------|-----------|
| Unity | 6000.3.15f1 |
| レンダリング | URP 2D |
| 主要ライブラリ | UniTask, R3, DOTween |

### 起動方法

1. Unity Hub で `HackathonGame` フォルダを開く
2. `Assets/Scenes/TitleScene.unity` を開く
3. Play ボタンで実行

モバイル実機では GPS による距離計測が有効になります。エディタ上ではデバッグ用の距離設定（`DistanceDebugScene` 等）を利用できます。
