# 開発・デバッグワークフロー

Cursor Agentは実装を行う際、以下のステップを例外なく実行してください。Quotaの無駄遣いを防ぎ、人間との連携をスムーズにすることが目的です。

## 1. ディレクトリ構成の確認
実装前に必ず `ProjectStructure.md` を参照し、正しい配置場所を確認すること。

## 2. 実装と出力のルール
- コードを出力する際は、変更がないメソッドは適宜省略し、修正箇所を中心に簡潔に出力してトークンを節約してください。

## 3. 【重要】Unityエディタ作業の人間への引き継ぎ
AIは `.unity` (Scene) や `.prefab` (Prefab) の直接編集が禁止されています。そのため、コードの実装が完了した際は、**必ず回答の最後に「Unityエディタでのセットアップ手順」を人間に対して明確に指示してください。**
- **指示すべき内容の例:**
  - どのスクリプトを、Hierarchy上のどのGameObject（またはPrefab）にアタッチするか。
  - Inspector上に公開された `[SerializeField]` に、何をアサイン（ドラッグ＆ドロップ）すべきか。
  - ScriptableObjectを新規作成した場合、エディタ上でアセットを作成する手順。

## 4. MCPを用いた自律デバッグフロー
実装完了後、人間からの指示を待たずに以下のセルフチェックを実行してください。しかしMCPツールを用いた操作はコンソールの読み取りのみ許可します。それ以外は禁止です。
1. **待機:** Unityがスクリプトのコンパイルを終えるまで数秒待機する。
2. **ログ取得:** MCPツールを使用して、Unityのコンソールログを取得する。
3. **エラー修正:** コンパイルエラーが出ている場合、即座に原因を分析して修正し、再度コンパイルとログ確認を行う。
4. **無限ループの回避:** エラー修正が3回以上連続で失敗した場合は行動を停止し、人間にエラー内容を報告して指示を仰ぐこと。
5. **完了報告:** エラーが0件であることを確認し、上記「3. 人間への作業指示」を出力して完了とする。
